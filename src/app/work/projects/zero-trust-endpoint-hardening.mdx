## What this was

A client engagement at Oncloud. The organization was running a perimeter-based model that hadn't kept up with their shift to remote work — unmanaged personal devices accessing corporate data, weak auth, no consistent security baseline across Windows, macOS, and mobile.

The goal: every device accessing M365 should be known, enrolled, compliant, and continuously verified.

## What I did

**Intune security baselines** — deployed Microsoft's baselines as a starting point, then layered custom CSPs for BitLocker enforcement and EDR sensor requirements. Built a compliance engine that automatically blocks M365 access for non-compliant devices.

**Conditional Access redesign** — audited the existing CA policies (there weren't many good ones), rebuilt from scratch. Enforced phishing-resistant MFA (FIDO2) and required a compliant device signal before accessing sensitive workloads. Took several iterations to get right without locking people out.

**Defender for Endpoint onboarding** — automated EDR onboarding across 10,000+ endpoints via Intune. Tuned alert configuration to reduce noise before handoff.

**Hardware attestation** — enforced TPM-based attestation and Secure Boot across the Windows fleet. Trickier with mixed hardware generations than it sounds.

**Exchange Online and MAM** — tightened mail-flow rules and deployed Mobile Application Management policies so corporate data couldn't leave via personal apps on unmanaged phones.

## What made it hard

Conditional Access is easy to get wrong in both directions. The first FIDO2 rollout broke access for users on older hardware that didn't support it. We ran a grace period with fallback MFA while hardware was replaced — unplanned but necessary.
